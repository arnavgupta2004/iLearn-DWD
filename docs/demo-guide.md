# Demo Guide

## 1. Best Demo Narrative

Present the project as an **AI-powered academic operating system**, not just a chatbot.

A strong framing line:

> iLearn DWD is a full-stack academic platform where AI supports course setup, tutoring, evaluation, analytics, and scheduling for both professors and students.

## 2. Recommended Demo Sequence

### Step 1. Professor Creates a Course

Show:

- syllabus upload
- Gemini-based extraction of course structure
- created course details

Talking point:

- This reduces course setup effort and turns unstructured syllabus PDFs into structured course data.

### Step 2. Professor Uploads Course Materials

Show:

- material upload
- course resources becoming available

Talking point:

- Uploaded materials become part of the AI tutor knowledge base through embeddings and retrieval.

### Step 3. Professor Creates Quiz or Assignment Request

Show:

- explicit professor action for `Create Quiz` or `Request Assignment`

Talking point:

- Students can only submit work after the professor creates the request, keeping evaluation professor-controlled.

### Step 4. Student Enrolls and Opens the Course

Show:

- enrolled course
- AI tutor inside the course page
- materials panel

Talking point:

- This is not a generic chatbot. It is a course-grounded tutor connected to uploaded materials.

### Step 5. Student Uses AI Tutor

Show:

- ask a course question
- AI response based on course context

Talking point:

- If the AI cannot answer confidently, the question can be flagged for the professor.

### Step 6. Student Completes a Quiz or Assignment

Show:

- quiz taking or assignment upload
- auto-generated score and feedback

Talking point:

- The platform handles evaluation and also feeds these results into long-term progress intelligence.

### Step 7. Student Opens To Do, Progress, and Calendar

Show:

- prioritized pending work
- course-wise progress breakdown
- personalized improvement advice
- interview request feature

Talking point:

- The platform helps students understand not just what is pending, but how to improve.

### Step 8. Professor Opens Analytics and Calendar

Show:

- course-wise analytics
- students strong in different topics
- objective completion and support suggestions
- interview requests

Talking point:

- Professors get actionable insights rather than just raw marks.

## 3. Strong Points to Emphasize

- role-based product design
- AI integrated into real workflows
- course-grounded tutoring using RAG
- automated evaluation
- personalized learning intelligence
- calendar and interview scheduling
- secure session-based validation in key APIs

## 4. If Asked “What Is Innovative Here?”

Suggested answer:

> The innovation is not a single AI feature, but the integration of multiple AI workflows into a complete academic system: syllabus parsing, material-grounded tutoring, automated evaluation, progress intelligence, topic-level analytics, and contextual page assistants.

## 5. If Asked “What Would You Build Next?”

Suggested answer:

- move advanced analytics into a Python microservice
- add background workers for heavy AI tasks
- improve study-buddy matching logic
- add predictive risk analytics for students falling behind
- extend calendar planning with recommendation logic

## 6. Submission Tip

If you submit screenshots or a presentation, the strongest pages to include are:

- professor analytics
- student progress
- student course chat
- professor flagged questions
- professor/student calendar
